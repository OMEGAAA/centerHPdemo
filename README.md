# スポーツ科学センター 公式サイト リニューアル提案

価格訴求中心ではなく **サービス内容の訴求** を重視した構成へ再設計したデモサイトです。
`index.html` をブラウザで開けば動作を確認できます。

---

## ① サイト全体構成（メニュー構成）

```
ホーム (index.html)
├ サービス（#services トップ内アンカー）
│   ├ 体力解析（service-body.html）           ← 旧スポーツドック
│   ├ 動作解析（service-motion.html）          ← 旧筋力解析の遷移先
│   ├ トレーニングジム（service-fitness.html） ← 旧アローズジム
│   └ ベースボールラボ（service-baseball.html）
├ アローズ（arrows.html）
│   ├ アローズラボ（arrows-lab.html）
│   └ アローズジム（arrows-gym.html）
├ 施設案内（facility.html）
├ 料金表（pricing.html）
├ アクセス（access.html）          ← 新規追加
├ ニュース（news.html）
├ イベント（events.html）
└ お問い合わせ（contact.html）
```

メニューバー：`サービス / アローズ / 施設案内 / 料金表 / アクセス / ニュース / お問い合わせ`

- 「ラボ・ジム」はまとめず、**アローズ** という独立メニューに整理
- **アクセス** を新規追加
- **タイアップ**はメニューから削除し、トップ下部へ配置
- **多目的室**は施設・料金から削除

## ② トップページ構成（セクション順）

1. ヘッダー（固定ナビ）
2. ヒーロー（科学で導く最高のパフォーマンス）
3. **イベントカレンダー & ピックアップニュース（最上部に配置）**
   - カレンダーは月切替可、当日と予定日をハイライト
   - ピックアップニュースはタブ切替で **同ページ内** で内容変更（ページ遷移なし）
4. 認定・活動情報（信頼感の訴求）
5. **サービスメニュー 2×2レイアウト**（回転アニメーション削除）
6. アローズ（ラボ／ジム）紹介ブロック
7. 実績（運営年数・利用者数）& タイアップ先
8. お問い合わせCTA
9. フッター

## ④ 導線改善案（少ないステップで詳細・問い合わせへ）

- ヘッダーに常時 **お問い合わせ（CTAボタン）** を表示
- すべてのページ末尾に **オレンジのCTAブロック** を設置（フォーム／電話／関連ページ）
- 料金表の各メニュー横に **「サービス詳細 →」直接リンク**
- contact.html では **電話／フォーム／来館** を3カードで並列提示（1画面で選べる）
- パンくずを全下層に配置し、戻り導線を確保

## ⑤ UI/UX改善ポイント（売上向上に直結）

| 改善 | 売上効果 |
|---|---|
| 価格より **サービス価値（測定項目・効果・対象）** を全面に | 「自分に必要」と感じさせ、申込率UP |
| イベント・ニュースを **トップ最上部** に配置 | リピート訪問増 + キャンペーン認知 |
| **認定・活動・実績** で信頼形成 | 高単価サービスの離脱率低下 |
| サービス2×2 + 回転アニメ削除 | 認知負荷を下げ、一覧性UP |
| ピックアップニュース **タブ切替（同ページ内）** | 滞在時間UP・直帰率低下 |
| 料金 → 詳細ページへ **直リンク** | 申込までのステップ削減 |
| 施設マップで **受付をパルスアニメで強調** | 来館初体験者の不安解消 |
| 一般客／会員／貸出／スタッフをエリアで色分け | 用途で迷わせない |
| ジュニア向け・40歳以上向けなど **ターゲット別の言葉** を併記 | 自分ごと化を促進 |

---

## 重視した3つの観点

1. **一般のお客様が使いやすいか** — 専門用語（スポーツドック・筋力解析）は併記/言い換え（体力解析・動作解析）。
2. **どのサービスを受けられるかすぐ分かるか** — 2×2のサービスカードで主力4つを即把握、各カードに対象・特徴・タグを表示。
3. **問い合わせしやすいか** — 全ページのヘッダーCTA + ページ末尾CTA + contact.html の3導線並列提示。

## 開発メモ

- 単一の `style.css` と `app.js` で全ページ共通制御
- カレンダーは JS で月送り対応、event オブジェクトを差し替えるだけで運用可能
- フォーム送信は現状ダミー（`alert`）— バックエンド連携時に差し替え
